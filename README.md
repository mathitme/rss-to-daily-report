# Daily Report

`Daily Report` is an open-source pipeline for exporting articles from Miniflux, storing them as Markdown, and generating AI-assisted daily digests.

`Daily Report` 是一个开源日报流水线，用于从 Miniflux 导出文章、保存为 Markdown，并生成 AI 辅助日报。

## What It Does

- Export Miniflux entries into a dated Markdown corpus.
- Generate a general digest from the recent article window.
- Generate a finance-themed digest from the same article corpus.
- Load prompt templates from `data/prompt` instead of hardcoding them in Python.
- Optionally send generated reports by SMTP.
- Run cleanly under cron with a portable runtime layout.

- 将 Miniflux 文章导出到按日期组织的 Markdown 语料库。
- 基于最近时间窗口内的文章生成综合日报。
- 基于同一份语料生成财经专题日报。
- 从 `data/prompt` 读取 prompt 模板，而不是在 Python 中硬编码。
- 可选通过 SMTP 发送生成后的日报。
- 采用可移植目录结构，便于放到 cron 中定时执行。

运行时目录说明：

- `data/md/`：正常运行时使用的文章 Markdown 存储目录。
- `data/md/examples/`：仓库内提交的示例语料，可用于本地联调和冒烟测试。
- `data/prompt/`：日报流水线使用的 prompt 模板目录。
- `data/report/`：生成后的日报输出目录。
- `data/log/`：脚本或 cron 运行时产生的日志目录。

## Requirements

- `python3`
- `pip`
- Miniflux server and API key if you want to export live entries
- An OpenAI-compatible LLM endpoint for digest generation
- Rust `html-to-markdown` CLI available in `PATH` if you want to export from Miniflux

依赖条件：

- `python3`
- `pip`
- 如果要导出真实订阅内容，需要可访问的 Miniflux 服务和 API key
- 如果要生成日报，需要兼容 OpenAI API 的模型服务
- 如果要从 Miniflux 导出文章，需要安装 Rust `html-to-markdown` 命令行工具并放入 `PATH`

## Installation

1. Clone the repository and install Python dependencies.

```bash
git clone https://github.com/mathitme/rss-to-daily-report.git
cd dailyReport
pip install -r requirements.txt
```

2. Prepare local config files.

```bash
cp .env.example .env
cp config/config.local.yaml.example config/config.local.yaml
```

3. Ensure the runtime directories exist.

```bash
mkdir -p data/md data/report data/log
```

4. Verify the HTML-to-Markdown converter if you plan to use Miniflux export.

```bash
html-to-markdown --version
```

安装步骤：

1. 克隆仓库并安装 Python 依赖。
2. 复制 `.env.example` 和 `config/config.local.yaml.example` 作为本地配置模板。
3. 创建运行时目录 `data/md`、`data/report`、`data/log`。
4. 如果要使用 Miniflux 导出功能，确认 `html-to-markdown` 已正确安装。

## Configuration

This repository uses two layers of configuration.

项目使用两层配置：

- Committed defaults in `config/config.yaml`
- Local private overrides in `config/config.local.yaml`

- 可提交的安全默认值放在 `config/config.yaml`
- 本地私密覆盖放在 `config/config.local.yaml`

`digest/parser.py` loads `config/config.yaml` first and then merges `config/config.local.yaml` on top of it.

`digest/parser.py` 会先加载 `config/config.yaml`，再将 `config/config.local.yaml` 合并覆盖进去。

### 1. `.env`

Use `.env` for runtime secrets and environment-level overrides.

`.env` 用于保存运行时密钥和环境级路径覆盖。

Example:

```dotenv
# Miniflux
MINIFLUX_URL=http://127.0.0.1:8080
MINIFLUX_API_KEY=your_miniflux_api_key_here

# Optional model env overrides
SILICONFLOW_API_KEY=your_siliconflow_api_key
LOCAL_API_KEY=your_local_api_key
LOCAL_BASE_URL=http://127.0.0.1:3003/v1

# Optional runtime path overrides
DAILYREPORT_DATA_DIR=/absolute/path/to/data
DAILYREPORT_MD_DIR=/absolute/path/to/data/md
DAILYREPORT_REPORT_DIR=/absolute/path/to/data/report
DAILYREPORT_PROMPT_DIR=/absolute/path/to/data/prompt
DAILYREPORT_LOG_DIR=/absolute/path/to/data/log

# Optional SMTP settings
DAILYREPORT_MAIL_RECIPIENT=your_email@example.com
DAILYREPORT_SMTP_HOST=smtp.example.com
DAILYREPORT_SMTP_PORT=587
DAILYREPORT_SMTP_USER=your_smtp_username
DAILYREPORT_SMTP_PASSWORD=your_smtp_password
DAILYREPORT_SMTP_FROM=your_email@example.com
DAILYREPORT_SMTP_USE_TLS=true
DAILYREPORT_SMTP_USE_SSL=false
```

Notes:

- `MINIFLUX_URL` and `MINIFLUX_API_KEY` are required only for live export from Miniflux.
- SMTP settings are optional. If they are missing, report generation still works and email sending is skipped.
- Runtime path overrides are optional. The public default layout is already `./data/md`, `./data/report`, `./data/prompt`, and `./data/log`.

说明：

- `MINIFLUX_URL` 和 `MINIFLUX_API_KEY` 只在导出真实 Miniflux 内容时必需。
- SMTP 配置是可选的。缺失时仍可生成日报，只是不会发送邮件。
- 路径覆盖也是可选的。公开版默认目录已经是 `./data/md`、`./data/report`、`./data/prompt`、`./data/log`。

### 2. `config/config.yaml`

`config/config.yaml` should stay safe to commit. It defines the default model wiring and can be used as the shared baseline for the project.

`config/config.yaml` 应始终保持可安全提交。它定义默认模型接线方式，可作为项目共享基线。

Current committed structure:

```yaml
llm:
  light:
    provider: "openai-compatible"
    api_key: ""
    base_url: "http://127.0.0.1:3003/v1"
    model: "small"

  heavy:
    provider: "openai-compatible"
    api_key: ""
    base_url: "http://127.0.0.1:3003/v1"
    model: "gpt"
    fallback_models: []
```

### 3. `config/config.local.yaml`

Put your real LLM credentials and any machine-specific overrides in `config/config.local.yaml`.

把真实模型密钥和机器相关覆盖写到 `config/config.local.yaml`。

Example:

```yaml
llm:
  light:
    api_key: "your_light_model_api_key"
    base_url: "http://127.0.0.1:3003/v1"
    model: "small"

  heavy:
    api_key: "your_heavy_model_api_key"
    base_url: "http://127.0.0.1:3003/v1"
    model: "gpt"
    fallback_models:
      - "kilo/auto-free"
```

`config/config.local.yaml` is ignored by git and must never be committed.

`config/config.local.yaml` 已被 git 忽略，不能提交。

## Prompt Templates

All digest prompts live in `data/prompt`.

所有日报 prompt 模板都位于 `data/prompt`。

This means you can tune summarization, filtering, and finance categorization behavior without editing Python code.

这意味着你可以直接调整筛选、归类、总结风格，而不需要修改 Python 代码。

## Usage

### A. Quick Smoke Test With Example Data

This is the easiest way to verify that the repo works without depending on your private Miniflux history.

这是最简单的验证方式，不依赖你的私有 Miniflux 历史数据。

Run the main digest against the committed examples:

```bash
./scripts/generate_digest.sh --date 2026-02-06 --md-dir ./data/md/examples
```

Run the finance digest against the committed examples:

```bash
./scripts/generate_finance_digest.sh --date 2026-02-06 --md-dir ./data/md/examples
```

Expected output:

- `data/report/YYYY/MM/YYYY-MM-DD.md`
- `data/report/YYYY/MM/YYYY-MM-DD-财经.md`

### B. Export Articles From Miniflux

Export unread entries:

```bash
./scripts/export_miniflux.sh
```

Export starred entries:

```bash
./scripts/export_miniflux.sh --status starred
```

Export all entries for a specific feed:

```bash
./scripts/export_miniflux.sh --status all --feed-id 65
```

List feeds only:

```bash
./scripts/export_miniflux.sh --list-feeds
```

Refresh one feed before export:

```bash
./scripts/export_miniflux.sh --refresh-feed 65 --mark-read
```

说明：

- 默认导出目录是 `data/md/`。
- 导出的文章会按 `YYYY/MM/DD/*.md` 目录结构保存。
- `--mark-read` 会在导出成功或按当前逻辑跳过后把 Miniflux 条目标记为已读。

### C. Generate the Main Digest

Generate for today:

```bash
./scripts/generate_digest.sh
```

Generate for a specific date:

```bash
./scripts/generate_digest.sh --date 2026-04-05
```

Generate from a custom article directory:

```bash
./scripts/generate_digest.sh --md-dir ./data/md/examples --date 2026-02-06
```

Useful options passed through to `digest/main.py`:

- `--date YYYY-MM-DD`
- `--md-dir /path/to/md`
- `--output-dir /path/to/report`
- `--anchor-time HH:MM`
- `--hours 36`
- `--test-limit N`

生成说明：

- 默认输出路径是 `data/report/YYYY/MM/YYYY-MM-DD.md`。
- 如果 SMTP 配置完整，脚本在生成完成后会自动调用 `digest/send_report.py` 发送邮件。
- 如果 SMTP 未配置，脚本只会生成文件，不会报错退出。

### D. Generate the Finance Digest

Generate for today:

```bash
./scripts/generate_finance_digest.sh
```

Generate for a specific date and article directory:

```bash
./scripts/generate_finance_digest.sh --date 2026-02-06 --md-dir ./data/md/examples
```

Useful options passed through to `digest/themed_main.py`:

- `--theme`
- `--theme-desc`
- `--date`
- `--md-dir`
- `--output-dir`
- `--anchor-time`
- `--hours`
- `--test-limit`

生成说明：

- 默认输出路径是 `data/report/YYYY/MM/YYYY-MM-DD-财经.md`。
- 默认主题已经由脚本封装为“财经”，但你也可以直接调用 Python 入口生成其他专题。

### E. Run Python Entry Points Directly

```bash
python -m miniflux.export_miniflux --url "$MINIFLUX_URL" --api-key "$MINIFLUX_API_KEY"
python -m digest.main --date 2026-04-05 --hours 36
python -m digest.main --date 2026-02-06 --md-dir ./data/md/examples
python -m digest.themed_main --theme "财经" --theme-desc "股票、基金、债券、宏观经济、货币政策、金融监管、市场行情等金融相关内容"
python -m digest.send_report --report ./data/report/2026/02/2026-02-06.md --subject "Daily Report 2026-02-06"
```

## Cron Setup

A ready-to-copy example is provided in `cron/daily-report.cron.example`.

仓库内提供了可直接参考的 cron 模板：`cron/daily-report.cron.example`。

Example:

```cron
*/30 * * * * cd /path/to/dailyReport && ./scripts/export_miniflux.sh >> ./data/log/miniflux-export-cron.log 2>&1
30 8 * * * cd /path/to/dailyReport && ./scripts/generate_digest.sh >> ./data/log/digest-cron.log 2>&1
50 8 * * * cd /path/to/dailyReport && ./scripts/generate_finance_digest.sh >> ./data/log/finance-digest-cron.log 2>&1
```

Before enabling cron:

- Confirm `.env` is present in the repo root used by cron.
- Confirm `python3` and `html-to-markdown` are available in that runtime environment.
- Run each script manually once before scheduling it.

在启用 cron 前：

- 确认 cron 使用的仓库根目录里存在 `.env`。
- 确认 `python3` 和 `html-to-markdown` 在该运行环境中可用。
- 先手动执行每个脚本一次，再加入定时任务。

## Publishing Notes

- Do not commit `.env`.
- Do not commit `config/config.local.yaml`.
- Do not commit generated files under `data/report/` or `data/log/`.
- Keep only reusable sample data under `data/md/examples/`.
- Do not publish any git history that already contains real secrets.

发布注意事项：

- 不要提交 `.env`。
- 不要提交 `config/config.local.yaml`。
- 不要提交 `data/report/` 和 `data/log/` 下的运行产物。
- `data/md/examples/` 只保留可复用的示例语料。
- 不要发布任何已经包含真实密钥的 git 历史。
